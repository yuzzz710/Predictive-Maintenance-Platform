数据探索与参数分析

14项统计任务 · 17组可视化 · 4参数诊断力评估
统计分析 ▶

技术方法

  • KDE分布拟合 — 正常/故障参数分布画像
  • Cohen's d + KS检验 — 14项任务量化区分力
  • Youden's J最优阈值搜索 — 单参数最佳切分点
  • 99%分位数安全区间 — 非参数运行边界
  • Pearson相关矩阵（正常 vs 故障）— 耦合变化检测
  • 衍生指标：视在功率(V×I)、效率(RPM/A)、热负荷(T/A)

关键发现

  • 所有4参数 Youden's J < 0.08（可用阈值 > 0.30）
  • 转速完全无诊断价值：Cohen's d = -0.002，KS p = 0.458
  • ~96-98%故障样本参数值落在正常范围内
  • 4参数在正常/故障态下均近乎独立（|r| < 0.09）

基线分析与统计告警系统

4基线模型 · 三级告警 · 逐设备μ/σ
统计基线 ▶

技术方法

  • 基线1: 逐设备复合Z-Score（V/A/T欧氏组合，独立μ/σ）
  • 基线2: 成本风险矩阵 Risk = P(fault)×UnitCost×DailyOutput
  • 基线3: 故障类型分层阈值（3组签名，不同z权重）
  • 基线4: Hotelling T²多元SPC（χ²控制限, p=3, α=0.01）
  • K-Means冷启动 — 稀疏数据设备借用聚类基线
  • 3层适配：集群回退 + 故障分组权重 + 产能分层阈值

关键发现

  • 设备间方差占61-73% — 全局阈值完全不可用
  • Z>2.0为推荐操作点：精确率83.9%，FPR=19.7%
  • T²精确率100%但召回率仅22% — 用作验证器
  • 约22%故障样本落在z<2.5范围内 — 静态基线不可见
