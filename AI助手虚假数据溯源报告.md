# AI助手虚假数据溯源报告

> 调查时间：2026-06-14  
> 触发问题：在设备矩阵界面问AI助手"为什么大多数设备都是退化和高危"，AI回答"健康率约92%，退化+高危约8%"——与实际情况严重不符。

---

## 一、结论先行

**AI助手给出的"健康率约92%"是完全虚假的数据。** 它不来自任何CSV数据文件，也不来自任何实时计算，而是直接写在系统提示词（Prompt）中的硬编码数字。AI被指令"必须用这些数字说话"，所以不管实际数据如何，都会输出这个虚假数字。

真实数据：
| 指标 | 实际值 | AI声称值 |
|------|--------|----------|
| 平均健康分 | **39.9** | ~92（声称"健康率92%"） |
| 危急设备(Critical) | **54台 (54%)** | 约3台 |
| 退化设备(Degrading) | **44台 (44%)** | 约8%（退化+高危合计） |
| 健康设备(>80分) | **0台 (0%)** | 约92台 |
| 日志故障比例 | **72.3%** (Types 1-9) | 未提及 |

---

## 二、虚假数据来源（三层溯源）

### 第1层（直接根因）：`routes.py` 的 `JUDGE_EXPLAIN_PROMPT`

**文件**：`web-dashboard\gateway\routes.py`  
**行号**：第 564 行

```python
JUDGE_EXPLAIN_PROMPT = """...
## 你的回答风格
- **用数字说话**：提到具体数据（健康率92%、3台危急、节费60-80%等）
..."""
```

**这是最直接的根因。** 当用户在设备矩阵界面（device-grid.html）提问时，如果话术库没有精确匹配（device-grid.html只有`__default__`），系统会走 AI fallback 路径（`/api/assistant/explain`）。该接口使用 `JUDGE_EXPLAIN_PROMPT` 作为 system prompt，其中**明确指令AI使用"健康率92%、3台危急"这些数字**。

AI忠实执行了这个指令——它是一台"照指令办事"的机器，不会质疑 prompt 中的数字是否真实。

### 完整调用链

```
用户在 device-grid.html 点击球体提问
  → assistant.js matchSpeech() 返回 __default__（无精确匹配）
  → matchType === 'default' → 触发 _renderAIFallback()
  → POST /api/assistant/explain
  → routes.py: assistant_explain() 
  → system_content = JUDGE_EXPLAIN_PROMPT
  → prompt 中写着："提到具体数据（健康率92%、3台危急、节费60-80%等）"
  → DeepSeek API 按照 prompt 指令生成回答
  → AI 输出："健康率约92%，退化+高危约8%"
```

### 第2层（间接来源）：`assistant.js` 话术库

**文件**：`web-dashboard\shared\assistant.js`  
**行号**：第 20 行（`home.html.__default__.speech.effect`）

```javascript
effect: "当前系统监控100台设备，其中健康率约92%，约3台处于危急状态。
系统已经自动为这些高危设备生成了维护工单..."
```

这个硬编码出现在 `home.html` 页面的默认话术中。如果用户在首页触发助手（不带自定义提问），会直接显示这段话。不过用户此次在 device-grid.html，所以这个话术并未被命中——但它说明"92%健康率"这个数字**在代码中多处硬编码**。

### 第3层（另一组矛盾数据）：`prompts.py` 系统提示词

**文件**：`web-dashboard\gateway\prompts.py`  
**行号**：第 186-187 行

```python
- 100台设备中10台健康分<40（危险），88台处于劣化状态
- 年停机420小时，年故障85次，停机成本$5000/h，故障成本$15000/次
```

这里的数据（10台<40，88台劣化）与 `JUDGE_EXPLAIN_PROMPT` 的"92%健康率"自相矛盾，但也与实际不符。实际是54台Critical + 44台Degrading。

## 六、为什么AI不说真数据？

因为 `/api/assistant/explain` 是**纯文本生成**路径，AI不做任何数据查询：
1. 没有 tool calling
2. 没有读取CSV文件
3. 没有查询数据库
4. 完全依赖 prompt 中的指令生成文本

Prompt 说"健康率92%"，AI就回答"健康率92%"——AI不知道这是假的。
