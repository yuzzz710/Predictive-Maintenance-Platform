# 智能设备预测性维护 — 项目文件树

> 100台CNC设备预测性维护系统。基于4个传感器参数（电压、电流、温度、转速），采用**成本-风险驱动 + 统计基线融合**的方式检测异常并生成维护工单。核心发现：4参数不足以支持纯ML预测（最大Youden's J=0.075），系统通过多信号融合（统计0.40 + 成本风险0.25 + ML密度0.25 + 趋势0.10）弥补信息不足。

```
│   ├── SKILL.md

│
├── predictive-maintenance-ml-inference/     # Skill 3: ML推理
│   ├── SKILL.md
│   └── scripts/
│       ├── run.py                           # CLI入口: --data-dir --prep-dir --output-dir --model
│       ├── model_training.py                # XGBoost v1: 6个模型(2窗口×3大小)+成本分层
│       └── model_training_v2.py             # MTNN v2: 6变体+鲁棒性分析
│
├── predictive-maintenance-diagnosis/        # Skill 4: 异常诊断
│   ├── SKILL.md
│   └── scripts/
│       ├── run.py                           # CLI入口: --data-dir --prep-dir --stat-dir [--ml-dir]
│       ├── maintenance_decision_engine.py   # 4种异常模式识别(电压漂移/热积聚/功率异常/综合退化)
│       └── predictability_analysis.py       # 5维度可预测性分析
│
└── predictive-maintenance-decision/         # Skill 5: 决策引擎
    ├── SKILL.md
    └── scripts/
        ├── run.py                           # CLI入口: --data-dir --prep-dir --stat-dir --max-orders

```

> **注意：** Skills共享代码采用**复制粘贴**方式（如 `maintenance_decision_engine.py`同时存在于diagnosis和decision skill中）。每个 `run.py`通过 `sys.path.insert(0, os.path.dirname(__file__))`导入本地模块。
